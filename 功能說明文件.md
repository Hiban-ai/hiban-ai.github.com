## 一、非技術人員：功能總覽

希絆雲作所是一個**多角色工作管理平台**，提供三種不同身份的使用者介面。

### 三種角色

| 角色 | 說明 |
|------|------|
| **工作夥伴**（partner） | 接案、搶單、回報工作、查看收入 |
| **督導人員**（supervisor） | 派案、審核工作回報、管理夥伴、查看統計 |
| **工作人員**（staff） | 帳號管理、薪資管理、公告發佈、系統設定 |

---

### 功能一：帳號與登入

- **線上申請加入**：新夥伴可線上填表申請，上傳身分證與存摺照片，系統自動寄通知信給管理人員審核。
- **AI 自動填表**：申請時可用相機拍身分證，系統透過 Google Gemini AI 自動辨識並填入姓名、身分證字號、生日、地址等資料，以及存摺帳號資訊。
- **登入保護**：Session 在當天午夜到期，需重新登入。首次登入強制修改密碼（預設密碼 `0000`）。
- **忘記密碼**：輸入姓名、身分證字號、生日送出申請，管理人員重設後密碼恢復為 `0000`。

---

### 功能二：任務管理

#### 派案（督導操作）
督導可建立兩種類型的任務：

1. **指派任務**：指定特定夥伴執行，夥伴可以接受或拒絕。
2. **搶單任務**：公開給所有夥伴搶，設定名額上限，先搶先得，系統防止超搶。

#### 夥伴接案流程
```
督導建立任務 → 夥伴收到通知 → 夥伴接受 → 執行任務 → 夥伴提交回報 → 督導審核通過 → 任務完成
```

- 拒絕任務時可填寫拒絕原因
- 搶單成功後自動建立進行中任務
- 任務有截止期限（預設 7 天）

---

### 功能三：工作回報與審核

- **夥伴提交**：填寫完成數量、附上連結與說明，最多可上傳 3 張截圖。
- **督導審核**：審核通過後任務標記完成；退回時可填寫原因，系統記錄在任務卡片上供夥伴參考，夥伴可重新提交。

---

### 功能四：薪資管理（工作人員）

- 按月檢視所有夥伴的已完成任務與收入統計。
- **匯出 Excel**：可選特定月份或全部，產生含總覽+個人分頁的試算表檔案。
- **寄送薪資通知信**：
  - 單獨寄給指定夥伴
  - 一次寄給全體夥伴
  - 寄彙整版本給管理員自己
- **每月自動寄信**：每月 1 日早上 8:00 自動寄送上個月薪資通知給所有有 Email 的夥伴。

---

### 功能五：公告系統

- 工作人員可發佈公告，指定對象（全部 / 工作夥伴 / 督導 / 工作人員）。
- 支援置頂公告、到期日設定（到期自動隱藏）。
- 可附加多個檔案（最大 15 MB），存放於 Google Drive，支援圖片預覽與下載。
- 各角色介面均有未讀提示，點開後標記已讀（記錄在瀏覽器 localStorage）。

---

### 功能六：問題回報

- 任何角色都可以提出問題，可選擇回報給「督導」或「管理員」。
- 支援圖片附件（最多 3 張）。
- 雙向對話：回報人與處理人可互相回覆。
- 有未讀訊息時顯示紅點徽章。
- 處理人可更新狀態：待處理 / 處理中 / 已解決。

---

### 功能七：帳號管理（工作人員）

- 審核新申請（核准時指定負責督導）、退回並填寫原因。
- 建立督導或工作人員帳號（勿需線上申請）。
- 停用帳號、刪除帳號（同步刪除 Google Drive 上的人員資料夾）。
- 重設密碼為 `0000`。
- 查看忘記密碼申請清單。
- 查看申請人上傳的身分證與存摺照片。

---

### 功能八：個人資料

- 所有角色均可修改：電話、地址、Email。
- 工作夥伴可額外更新銀行（或郵局）帳戶資料。
- 修改密碼（需輸入現有密碼，新密碼 6～12 位）。

---

### 功能九：雲端整合

| 整合服務 | 用途 |
|----------|------|
| **Firebase Firestore** | 主資料庫（任務、搶單、回報、公告等） |
| **Google Drive** | 儲存人員文件照片、公告附件 |
| **Google Gemini AI** | 身分證與存摺圖片 OCR 自動填表 |
| **Google Apps Script** | Email 寄送服務（薪資通知、審核結果） |

| 函式 / 物件 | 說明 |
|-------------|------|
| `API.get(url)` | 封裝 fetch GET，自動拋出錯誤 |
| `API.post(url, body)` | 封裝 fetch POST |
| `API.put(url, body)` | 封裝 fetch PUT |
| `API.delete(url)` | 封裝 fetch DELETE |
| `requireAuth(roles[])` | 驗證登入與角色，未符合導回 `/index.html` |

| `getReadIds(userId)` | 從 localStorage 取已讀公告 ID 陣列 |
| `markRead(userId, id)` | 標記單一公告已讀 |
| `markAllRead(userId, ids)` | 標記全部公告已讀 |

### 後端主要模組

| 檔案 | 說明 |
|------|------|
| `server.js` | Express 主程式，所有 API 路由與業務邏輯 |

| `shared.js` | 前端共用 JS |
| `firebase-key.json` | Firebase 服務帳號金鑰（勿提交版本控制） |

## 三、開發者 API 完整說明

> 所有 API 以 `/api/` 為前綴，回傳 JSON。  
> 認證透過 express-session Cookie。

| `POST` | `/api/logout` | 已登入 | 登出，銷毀 Session |
| `GET` | `/api/me` | 已登入 | 取得目前登入者資訊（含 `login_dates`） |
| `POST` | `/api/register` | 公開 | 申請帳號，回傳系統產生的 `username` |
| `POST` | `/api/forgot-password` | 公開 | 送出忘記密碼申請（比對姓名+身分證+生日） |
| `POST` | `/api/change-password` | 已登入 | 修改密碼（需提供舊密碼，新密碼 6～12 位） |

### 個人資料

| 方法 | 路徑 | 權限 | 說明 |
